# Mock 数据测试指南

> ✨ **重要更新**：Mock 系统已升级为**动态数据**，支持真实的增删改查！

---

## 🎯 测试目的

在后端 API 未完成前，使用 Mock 数据模拟真实的 API 响应，测试前端功能完整性。

---

## ⚙️ 配置说明

### 1. 开启 Mock 模式

文件：`.env.development`

```bash
# Mock开关（true=使用假数据，false=调用真实API）
VITE_USE_MOCK=true  # ← 确保为 true
```

### 2. 重启开发服务器

⚠️ **重要**：修改环境变量后必须重启！

```bash
Ctrl+C  # 停止当前服务
npm run dev  # 重新启动
```

## ✅ 新 Mock 系统特性

### 动态数据存储

与旧系统的静态数据不同，新系统支持：

| 功能     | 旧 Mock                     | 新 Mock                     |
| -------- | --------------------------- | --------------------------- |
| 提交日记 | ❌ 只返回成功，不添加到列表 | ✅ 真正添加到列表开头       |
| 删除日记 | ❌ 只返回成功，列表不变     | ✅ 真正从列表中删除         |
| 查看详情 | ❌ 始终返回固定数据         | ✅ 从动态列表中查找         |
| 数据持久 | ❌ 刷新页面后丢失           | ⚠️ 仅会话级持久（刷新丢失） |

## 🧪 测试步骤

### 测试 1：提交日记后列表自动刷新

1. **打开页面**：访问 `/mood-diary`
2. **查看初始列表**：应该有 8 条 Mock 日记
3. **填写表单**：
   - 选择情绪：任选一个（如"开心"）
   - 情绪强度：拖动滑块（如 7 分）
   - 内容：随便写点什么
4. **提交**：点击"提交日记"按钮
5. **验证结果**：
   - ✅ 显示"日记提交成功"提示
   - ✅ 显示 AI 分析弹窗
   - ✅ 表单自动重置
   - ✅ **列表顶部出现刚提交的日记**（⭐ 新功能）
   - ✅ 总数变为 9 条

### 测试 2：删除日记功能

1. **点击任意日记卡片**：进入详情页
2. **点击删除按钮**：在详情页点击"删除"
3. **确认删除**：在弹窗中确认
4. **验证结果**：
   - ✅ 显示"删除成功"提示
   - ✅ 自动跳转回列表页
   - ✅ **该日记已从列表中消失**（⭐ 新功能）
   - ✅ 总数减 1

### 测试 3：新提交的日记可查看详情

1. **提交一篇新日记**（参考测试 1）
2. **点击刚提交的日记卡片**
3. **验证结果**：
   - ✅ 能正常打开详情页
   - ✅ 显示刚才填写的内容
   - ✅ 显示模拟的 AI 分析
   - ✅ 情绪类型、分数正确

## 🔍 Mock 数据详情

### 初始数据

```javascript
初始日记数量：8条
日期范围：2024-11-20 ~ 2024-11-27
情绪类型：Happy, Calm, Anxious, Excited, Tired, Sad
```

### 新增日记规则

```javascript
ID生成：自增（从5009开始）
Emoji：根据情绪类型自动匹配
位置：添加到列表开头（最新优先）
AI分析：固定模拟文案
```

### 数据持久性

⚠️ **重要限制**：

- **会话级持久**：在当前浏览器会话中有效
- **刷新页面**：数据恢复到初始状态（8 条）
- **不同标签页**：数据不共享
- **实际后端**：连接真实 API 后，数据将永久存储

## 🐛 已知限制

### 1. 数据不持久化

```
问题：刷新页面后，新增的日记会丢失
原因：Mock数据存储在内存中
解决：等待后端完成，使用真实数据库
```

### 2. 趋势图数据不动态

```
问题：提交日记后，首页趋势图不更新
原因：趋势图使用独立的静态Mock数据
影响：不影响日记列表功能测试
解决：后端完成后自动解决
```

### 3. AI 分析内容固定

```
问题：所有日记的AI分析都一样
原因：Mock数据返回固定文案
影响：可测试UI展示，但无法测试AI逻辑
解决：等待后端AI接口
```

## 🔄 切换到真实 API

后端完成后，按以下步骤切换：

### 1. 修改环境变量

```bash
# .env.development
VITE_API_BASE_URL=http://your-backend-url/api  # 改为后端实际地址
VITE_USE_MOCK=false  # 关闭Mock
```

### 2. 重启服务

```bash
Ctrl+C
npm run dev
```

### 3. 验证对接

- 打开浏览器开发者工具 → Network
- 提交日记
- 检查请求是否发送到真实后端
- 检查响应数据格式

## 📞 问题反馈

### Mock 功能异常

1. 确认 `VITE_USE_MOCK=true`
2. 确认已重启开发服务器
3. 打开浏览器控制台查看错误

### 数据不刷新

1. 检查 API 调用是否成功（Network 面板）
2. 检查控制台是否有错误
3. 尝试刷新页面
