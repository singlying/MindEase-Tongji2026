# 医生端&管理员端前端开发说明

## 1. 当前开发现状与启动方式

- 项目路径：`mindease-frontend/`，Vite + Vue3 + TypeScript + Element Plus + Tailwind。
- 启动（联调或 mock 通过 `VITE_USE_MOCK` 环境变量切换）：
  - 安装依赖：`npm install`
  - 开发启动：`npm run dev`（默认读取 `.env.*`；`VITE_USE_MOCK=true` 走前端 mock，false 走后端）
  - 构建：`npm run build`
- 用户端功能已基本完成（认证、推荐/预约、情绪报告、通知等），医生端/管理员端尚未开发页面与路由。

## 2. 文件/路由建议

- 医生端（咨询师端）页面建议放在：`src/views/counselor/` 下新增：
  - `ScheduleView.vue`：排班设置/更新。
  - `AppointmentsManageView.vue`：查看/确认预约列表与详情。
  - 如需复用审核状态，可在现有 `AuditPendingView.vue` 基础上增加“审核结果查看/跳转到排班”指引，而非重复实现提交逻辑。
- 管理员端页面建议放在：`src/views/admin/` 下新增：
  - `AuditListView.vue`：资质审核列表。
  - `AuditProcessView.vue`（或在列表内弹窗）：审核通过/驳回操作。
  - 如后续有量表管理，可在此目录再扩展（但当前接口仅涉及审核）。
- 路由：在 `src/router` 中新增对应路由；鉴权建议基于用户角色（`user`/`counselor`/`admin`）。可沿用现有路由守卫逻辑（检查 token 与角色）。

## 3. 设计风格参考

- 参考 `html_example` 目录的原型样式，与现有用户端保持一致的 glass / 卡片风格。
- 公共样式：`src/assets/main.css`、`shared-styles-4.css` 已融入的变量、色彩、玻璃卡片。组件尽量简洁，复用已有的按钮/卡片样式。

## 4. 医生端（咨询师端）需实现的功能（仅列有接口支撑的功能）

1. 资质审核状态查看
   - 接口：`GET /counselor/audit/status`
   - 展示最新状态（PENDING/APPROVED/REJECTED）与备注；若未提交，引导跳转提交页（可沿用现有 `AuditPendingView`）。
2. 设置/更新排班
   - 接口：`POST /appointment/schedule`
   - 表单字段：`workDays`（数字数组 1-7），`workHours`（时间段数组）。提交成功提示“排班设置成功”。
3. 查询可用时段（用于自检排班正确性，可选）
   - 接口：`GET /appointment/available-slots`，参数：`counselorId`、`date`。
   - 主要用于调试或预览，不必作为核心入口。
4. 预约管理
   - 列表：`GET /appointment/my-appointments`（在咨询师身份下，`targetRole` 为 user）。支持按状态筛选（PENDING/CONFIRMED/CANCELLED 等后端定义）、分页。
   - 详情：`GET /appointment/{id}`。
   - 确认预约：`PUT /appointment/{id}/confirm`。
   - 取消预约：`PUT /appointment/{id}/cancel`（需填写取消原因）。
   - UI 建议：列表卡片 + 详情抽屉/对话框，提供“确认”“取消”操作按钮。

## 5. 管理员端需实现的功能（仅列有接口支撑的功能）

1. 资质审核列表
   - 接口：`GET /admin/audit/list`，参数：`page`、`pageSize`。展示：`auditId`、`userId`、`username`、`realName`、`qualificationUrl`、`submitTime` 等。
2. 审核操作
   - 接口：`POST /admin/audit/process`，请求体：`auditId`、`action` (`PASS`/`REJECT`)、`remark`。
   - UI 建议：列表行内或详情弹窗中操作。通过后提示“资质审核通过”；驳回需填写 remark。

## 6. 需要改动/新建的页面位置

- 新建文件：
  - `src/views/counselor/ScheduleView.vue`
  - `src/views/counselor/AppointmentsManageView.vue`

## 7. 后端接口清单（医生端/管理员端需用）

- 认证/个人信息：`POST /auth/login`、`GET /auth/profile`（用于获取角色、昵称、头像），`PUT /auth/profile`（仅如需改昵称/头像）。
- 咨询师审核：`POST /counselor/audit/submit`（若医生端需要重新提交）、`GET /counselor/audit/status`。
- 排班与预约（咨询师侧）：
  - `POST /appointment/schedule`
  - `GET /appointment/available-slots`
  - `GET /appointment/my-appointments`
  - `GET /appointment/{id}`
  - `PUT /appointment/{id}/confirm`
  - `PUT /appointment/{id}/cancel`
- 管理员审核：
  - `GET /admin/audit/list`
  - `POST /admin/audit/process`

## 8. 其他注意事项

- 角色鉴权：进入医生端需 role=counselor，管理员端需 role=admin；未授权应重定向登录或无权限页。
- Token 处理：沿用现有 axios 拦截器；管理员/医生登录后需确保 token 存储并在刷新后恢复。
- 上传：当前接口未提供通用上传，资质图片字段使用 URL（可先支持输入/粘贴 URL）。
- 文案与交互：保持与用户端一致的提示与按钮风格，表单校验规则与用户端现有实现一致（必填项：真实姓名、资格证书、职称、从业年限、专长、价格、所在地）。
- Mock：如需 mock 支撑医生/管理员端调试，可在 `src/mock/` 下补充对应接口数据并在 `VITE_USE_MOCK=true` 时启用，保持与真实接口字段一致。
