## 2. 项目概述

MindEase 是一个面向三类用户（普通用户、咨询师、管理员）的心理健康服务平台后端，共包含 **10个功能模块**、**13张数据表**、**约50个API接口**。

### 2.1 功能模块清单

| 编号 | 模块 | 路由前缀 | 核心功能 |
|------|------|---------|---------|
| M1 | 用户认证与权限 | `/auth`, `/admin/common` | 注册（user/counselor角色）、登录、JWT鉴权、个人信息管理 |
| M2 | 用户中心 | `/user` | Dashboard数据聚合、系统通知管理 |
| M3 | 咨询师资质审核 | `/counselor/audit`, `/admin/audit` | 资质提交、审核状态查询、管理员审核操作 |
| M4 | 咨询师推荐 | `/counselor` | 智能推荐、详情展示、评价系统 |
| M5 | 预约管理 | `/appointment` | 排班设置、时段查询、创建/取消/确认预约 |
| M6 | 心理测评 | `/assessment`, `/admin/assessment` | 量表管理、提交测评、自动评分、历史查询 |
| M7 | 情绪日记 | `/mood` | 日记增删查、情绪趋势、统计数据 |
| M8 | AI 咨询 | `/chat` | 会话管理、流式消息、敏感词检测、ASR/TTS |
| M9 | 情绪报告 | `/report` | 报告生成、PDF导出 |
| M10 | 文件上传 | `/upload` | 阿里云OSS文件上传 |

### 2.2 关键技术组件

| 组件 | 技术选型 |
|------|---------|
| 数据库 | MySQL 8.x |
| 缓存/向量存储 | Redis |
| ORM | MyBatis 3.0.4（注解 + XML混合） |
| 鉴权 | JWT (HMAC-SHA256, 0.12.6) |
| 加密 | AES-256-GCM 敏感字段透明加密 |
| AI对话 | LangChain4j + 通义千问 (qwen-max) |
| RAG | Redis 向量存储 + 知识库检索 |
| 语音 | 阿里云 ASR (Paraformer-v2) + TTS (Sambert) |
| 文件存储 | 阿里云 OSS |
| PDF生成 | iText 5.5 + JFreeChart |
| API文档 | Knife4j 4.3.0 |

## 3. 测试策略

### 3.1 测试金字塔

```
           ┌──────────┐
           │  E2E 测试 │  ← 全链路场景验证（少量）
           ├──────────┤
           │ 集成测试  │  ← API接口 + 数据库 + Redis（适量）
           ├──────────┤
           │ 单元测试  │  ← Service/Utils/异常/常量（大量）
           └──────────┘
```

### 3.2 测试层级定义

| 层级 | 范围 | 占比目标 | 说明 |
|------|------|---------|------|
| **L1 单元测试** | 工具类、异常类、常量类、DTO/VO、Service 纯逻辑方法 | 60% | 不依赖Spring容器，快速验证 |
| **L2 集成测试** | Controller → Service → Mapper 完整链路、数据库操作 | 30% | 依赖Spring容器和真实数据库 |
| **L3 E2E测试** | 多接口串联的业务流程 | 10% | 完整业务场景验证 |

## 6. L3 E2E 场景测试

### 6.1 用户完整旅程

```
场景E2E-01：新用户首次使用 → 完成测评 → AI咨询 → 情绪日记 → 获取推荐

步骤：
  1. POST /auth/register  注册普通用户
  2. POST /auth/login  登录获取Token
  3. GET  /assessment/scales  浏览量表列表
  4. GET  /assessment/scale/{key}  选择量表
  5. POST /assessment/submit  完成测评
  6. POST /chat/session  创建AI会话
  7. POST /chat/message  进行AI对话
  8. POST /mood/log  记录情绪日记
  9. GET  /mood/trend  查看情绪趋势
  10. GET /counselor/recommend  获取咨询师推荐
```

### 6.2 咨询师注册审核旅程

```
场景E2E-02：咨询师注册 → 资质审核 → 排班 → 接诊

步骤：
  1. POST /auth/register  role=counselor  注册咨询师
  2. POST /auth/login  登录（PENDING状态，访问受限）
  3. POST /counselor/audit/submit  提交资质材料
  4. POST /admin/common/login  管理员登录
  5. GET  /admin/audit/list  查看审核列表
  6. POST /admin/audit/process  action=APPROVE  审核通过
  7. POST /auth/login  咨询师重新登录（已可正常访问）
  8. POST /appointment/schedule  设置排班
```

## 10. 风险与注意事项

### 10.1 测试风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| AI服务不可用 | M8模块无法测试 | 使用Mock/降级配置验证 |
| 阿里云服务依赖 | ASR/TTS/OSS测试受阻 | 优先Mock，条件允许时真实测试 |
| 测试数据准备不全 | 部分用例无法执行 | 提前编写数据初始化SQL脚本 |
| Redis不可用 | AI对话记忆功能测试受阻 | 本地安装Redis或使用Testcontainers |

### 10.2 注意事项

1. **测试数据库隔离**：必须使用独立测试数据库（`mindease_test`），不得与开发/生产数据库共用
2. **AES密钥一致性**：测试环境的`MINDEASE_SECURITY_AES_KEY`必须与数据库加密数据匹配
3. **JWT密钥有效期**：单元测试使用较短的TTL（如60秒）以测试过期场景
4. **敏感词检测**：不测试真实心理危机场景（如自杀、自残），使用中性文本验证过滤逻辑
5. **Mock外部服务**：阿里云OSS、DashScope等外部依赖在测试中优先使用Mock，避免产生费用
