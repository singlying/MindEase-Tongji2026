## 3. JWT 认证安全测试

### 3.1 JWT 配置检查

```yaml
# 当前配置
mindease:
  jwt:
    secret-key: ${MINDEASE_JWT_SECRET_KEY:}  # ⚠️ 若未设置则为空字符
    ttl: 7200000                              # 2小时
```

**安全风险：** 如未配置 `MINDEASE_JWT_SECRET_KEY` 环境变量，密钥为空字符串。

### 3.2 JWT 安全测试用例

| 编号 | 用例名称 | 攻击方式 | 预期结果 |
|------|---------|---------|---------|
| TC-JWT-01 | 无Token访问 | 不带 header 访问受保护接口 | 401 Unauthorized |
| TC-JWT-02 | 空Token | header `token:` 为空 | 401 Unauthorized |
| TC-JWT-03 | 过期Token | 使用已过期的JWT（>2小时） | 401 Unauthorized |
| TC-JWT-04 | 伪造Token | 用不同secretKey签名的Token | 401 Unauthorized |
| TC-JWT-05 | 篡改Payload | 修改Token中的userId为其他用户 | 401 或 签名校验失败 |
| TC-JWT-06 | 篡改角色 | 将role从user改为admin | 签名校验失败 |
| TC-JWT-07 | None算法 | JWT使用`"alg":"none"` | 被JJWT库拒绝 |
| TC-JWT-08 | HS256→HS512变体 | 尝试不同签名算法 | 被拒绝 |
| TC-JWT-09 | Token重复使用 | 同一Token多次请求 | 正常使用（无单次Token机制） |
| TC-JWT-10 | Token泄露日志 | 检查日志是否打印Token | Token不应打印到日志 |
| TC-JWT-11 | Redis不可用时JWT | 关闭Redis后请求 | JWT不依赖Redis，正常使用 |

### 3.3 JWT 测试代码

```java
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
@AutoConfigureWebTestClient
class JwtSecurityTest {

    @Test
    void testExpiredToken() {
        // 使用极短TTL生成Token，等待过期后测试
        String token = generateTokenWithTtl(1); // 1ms TTL
        Thread.sleep(100);
        
        webTestClient.get()
            .uri("/auth/profile")
            .header("token", token)
            .exchange()
            .expectStatus().isEqualTo(401);
    }

    @Test
    void testForgedToken() {
        // 用不同密钥生成Token
        String forgedToken = JwtUtil.createJWT(
            "different-secret-key-32bytes!!", // 不同密钥
            7200000,
            Map.of("userId", "1", "role", "admin")
        );
        
        webTestClient.get()
            .uri("/auth/profile")
            .header("token", forgedToken)
            .exchange()
            .expectStatus().isEqualTo(401);
    }

    @Test
    void testTamperedPayload_NoneAlgorithm() {
        // JJWT 默认不支持 none 算法
        // 尝试手动构造 none 算法JWT
        String noneJwt = "eyJhbGciOiJub25lIn0.eyJ1c2VySWQiOiIxIiwicm9sZSI6ImFkbWluIn0.";
        
        webTestClient.get()
            .uri("/auth/profile")
            .header("token", noneJwt)
            .exchange()
            .expectStatus().isEqualTo(401);
    }
}
```

### 5.2 密码安全

```java
// 当前使用MD5（UserServiceImpl.login()）
import org.springframework.util.DigestUtils;
String encryptedPassword = DigestUtils.md5DigestAsHex(password.getBytes());
```

**⚠️ 安全风险：** 
- MD5 已被破解，存在碰撞攻击
- 无盐值（Salt），彩虹表攻击风险
- 建议升级为 BCrypt

**测试用例：**

| 编号 | 用例名称 | 说明 | 状态 |
|------|---------|------|------|
| TC-PWD-01 | 密码非明文存储 | 数据库中password为加密哈希 | ✅ MD5 |
| TC-PWD-02 | 弱密码检测 | 注册时验证密码长度和复杂度 | ⬜ 待验证 |
| TC-PWD-03 | 彩虹表攻击 | 常见密码"123456"碰撞 | ⬜ 建议升级BCrypt |
| TC-PWD-04 | 暴力破解防御 | 连续失败登录检测 | ⬜ 建议增加 |
| TC-PWD-05 | 密码传输 | 是否HTTPS加密传输 | ⬜ 待确认 |

## 6. 注入攻击测试

### 6.1 SQL注入

**当前防护：** MyBatis 使用 XML Mapper 参数化查询 (`#{参数}`)，天然防注入。

**测试用例（重点测试动态SQL场景）：**

| 编号 | 用例名称 | 注入Payload | URL |
|------|---------|------------|-----|
| TC-SQLI-01 | 登录SQL注入 | `' OR '1'='1` | POST `/auth/login` |
| TC-SQLI-02 | 搜索注入 | `'; DROP TABLE sys_user; --` | GET `/counselor/recommend?keyword=` |
| TC-SQLI-03 | LIKE注入 | `%' OR 1=1 --` | 含LIKE的查询接口 |
| TC-SQLI-04 | 数字型注入 | `1 OR 1=1` | GET `/appointment/{id}` |
| TC-SQLI-05 | 联合查询 | `' UNION SELECT 1,2,3 --` | 登录接口 |
| TC-SQLI-06 | 时间盲注 | `' AND (SELECT SLEEP(5)) --` | 任一带参查询接口 |
| TC-SQLI-07 | MyBatis ${}安全 | 检查XML是否使用`${}`动态SQL | 代码审查 |

### 6.2 XSS 测试

| 编号 | 用例名称 | 输入 | 注入点 |
|------|---------|------|--------|
| TC-XSS-01 | 基本XSS | `<script>alert(1)</script>` | 情绪日记content |
| TC-XSS-02 | IMG XSS | `<img src=x onerror=alert(1)>` | 聊天消息content |
| TC-XSS-03 | SVG XSS | `<svg onload=alert(1)>` | 用户昵称 |
| TC-XSS-04 | 事件处理XSS | `<div onmouseover=alert(1)>` | 预约备注 |
| TC-XSS-05 | JavaScript URI | `javascript:alert(1)` | 头像URL |
| TC-XSS-06 | 编码绕过 | `%3Cscript%3Ealert(1)%3C/script%3E` | content |

**⚠️ 注意：** MindEase 是 API 服务，XSS 风险主要在**前端展示**阶段。后端应在输出时确保数据被正确转义。当前使用 AES 加密存储，数据完整性有保护但无输出编码。

## 11. 安全配置检查清单

### 11.1 application.yml 安全检查

```yaml
# ✅ 良好实践
mindease:
  jwt:
    secret-key: ${MINDEASE_JWT_SECRET_KEY:}  # 使用环境变量
  security:
    aes-key: ${MINDEASE_SECURITY_AES_KEY:}    # 使用环境变量

# ❌ 风险项
# - secret-key 默认值为空字符串
# - aes-key 默认值为空字符串时使用硬编码密钥
# - DASHSCOPE_API_KEY 未配置时接口暴露
# - CORS 允许 localhost (生产需改为域名)
```

### 11.2 生产环境安全检查

| 检查项 | 状态 | 建议 |
|--------|------|------|
| HTTPS 强制 | ⬜ | 生产必须启用TLS |
| 安全响应头 | ⬜ | 添加 X-Frame-Options, X-Content-Type-Options 等 |
| 错误页面定制 | ✅ | GlobalExceptionHandler 统一处理 |
| 日志脱敏 | ⬜ | 密码、Token不应记录 |
| 会话管理 | ✅ | JWT无状态，无需Session |
| 文件上传限制 | ✅ | max-file-size: 10MB |
| API限流 | ⬜ | 建议增加 |
| 健康检查保护 | ⬜ | Actuator端点需鉴权 |

### 11.3 安全加固建议

## 12. 密码策略建议

### 12.1 当前状态

| 项目 | 当前 | 建议 |
|------|------|------|
| 哈希算法 | MD5 (无盐) | BCrypt (带盐, cost=10) |
| 最小长度 | 无限制 | ≥8字符 |
| 复杂度要求 | 无 | 至少1大写+1数字+1特殊字符 |
| 失败锁定 | 无 | 5次失败锁定15分钟 |
| 密码历史 | 无 | 不可与最近3次相同 |

### 12.2 BCrypt 升级方案

```java
// 替换 UserServiceImpl 中的密码加密
// 现有:
import org.springframework.util.DigestUtils;
String password = DigestUtils.md5DigestAsHex(user.getPassword().getBytes());

// 建议:
import org.springframework.security.crypto.bcrypt.BCryptPasswordEncoder;
private final BCryptPasswordEncoder encoder = new BCryptPasswordEncoder(10);
String password = encoder.encode(user.getPassword());

// 验证:
boolean matches = encoder.matches(rawPassword, storedHash);
```
